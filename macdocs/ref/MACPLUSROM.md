## Boot Sequence (quick outline)

1. **`$2A` EBootStart** → Check test software, branch to L1
2. **`$62` L3** → Set SR to supervisor, read VIA timing, init VIA registers
3. **`$8E`** → Init VIA register B direction/value, overlay off
4. **`$FC`** → Init IWM
5. **`$1EA` P_EraseScrnBuff** → Clear screen buffer
6. **`$28A` P_mBootBeep** → Startup sound
7. **`$352` P_BootPart2** → Fill RAM with `$FFFFFFFF`, call `P_mSysErrInit`
8. **`$368`** → Second test-software check
9. **`$370`** → Init globals (`VIA`, `IWM`, `SoundBase`, `ScreenBase`), cursor data
10. **`$452`–`$57E`** → Set up A-line trap dispatch tables (`DispatchTab` + `ToolDispTable`)
11. **`$580`–`$5B4`** → Determine CPU type (`CPUFlag`)
12. **`$5B6`–`$616`** → Init SCC, I/O manager, install disk driver, start SCSI
13. **`$6F4` P_WaitForBootDisk** → Question-mark disk icon loop
14. **`$986` P_BootFromDisk** → Read boot blocks, jump to system startup

## How to Use This Reference

### Finding a trap implementation
1. Look up the trap name in `TRAP_TABLE.md` to get the trap word
2. Search this document for the `P_t<Name>` procedure
3. Note the ROM address — in the listing it appears as the first hex field

### Setting a breakpoint on a ROM routine
Use the ROM address + `$400000` (ROM base on Mac Plus):
- Example: `P_tGetResource` at `$13DEE` → absolute address `$413DEE`

### Tracing hardware interaction
The ROM accesses hardware at these bases:
- VIA: `$EFE1FE` (register B) — search listing for `EFE1FE`
- IWM: `$DFE1FF` — search for `DFE1FF`
- SCC: `$9FFFF8` — search for `9FFFF`
- SCSI: `$580000`/`$580001` — search for `580000`

### Finding the line in the listing
Use `grep -n 'P_<name>' macdocs/ref/plus-rom-listing.asm.txt` to find the
exact line, then `read_file` with that line range.

### Naming convention
- `P_t<Name>` — Toolbox/OS trap implementation (the `t` prefix)
- `P_m<Name>` — Internal/private ROM routine (the `m` prefix)
- `P_<Name>` — Other labeled routine (device-specific, etc.)
- `L<N>` — Local label (unnamed branch target)
- `E_<N>` — Continuation label (return-from-subroutine target)
- `DT<Name>` — Data table

### Cross-referencing with mac-rom sources
The `macdocs/ref/mac-rom/` directory contains original Apple ROM source code
(from the "SuperMario" sources). Key directories:
- `OS/` — Operating system: memory manager, file system, drivers
- `Toolbox/` — Window/Menu/Dialog/Control/TextEdit managers
- `QuickDraw/` — Graphics subsystem
- `Drivers/` — Device drivers
- `Patches/` — ROM patches applied by System file
