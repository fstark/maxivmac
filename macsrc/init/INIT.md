# maxivmac Guest INIT

**maxivmac INIT** — resource type `INIT`, ID 128.

A single INIT resource that provides all host-integration services:
shared drive mounting, clipboard sync, and guest-command polling.
A jGNEFilter handles all periodic tasks.

## Source Files

| File | Responsibility |
|------|---------------|
| `init.c` | INIT entry (`main`), jGNEFilter (`FilterEntry`), dynamic 68k trap stub generation, boot sequence |
| `clip.c` | Clipboard sync: `ExportMacToHost`, `ImportHostToMac`, `SyncClipboard` (throttled to every 30 ticks) |
| `drive.c` | Shared drive File Manager trap handlers, FCB management, VCB/DQE allocation, dispatch tables, `MountNewDrive` |
| `comm.c` | Extension discovery (`find_reg_base`), register I/O (`reg_set`/`reg_get`/`reg_command`/`reg_result`), debug logging, host KV store access, utility functions |
| `defs.h` | All constants (low-memory addresses, PB/FCB offsets, error codes, command codes), `Globals` struct, prototypes |
| `version.h` | Auto-generated by `build-init.sh` — embeds git-derived version string |

## Globals

Allocated in the system heap via `NewPtrSysClear`. The pointer is
stored on the host side via `ExtFSGuestVars` ($020E) so it survives
application context switches.

```c
typedef struct {
    char   *regBase;           /* extension register base pointer */
    long    savedA4;           /* THINK C A4 for code resource relocation */
    Ptr     vcb[kMaxDrives];   /* VCB pointers, up to 8 drives */
    Ptr     dqe[kMaxDrives];   /* drive queue element blocks */
    short   driveCount;
    long    volFileCount;
    long    volTotalBytes;
    short   ejected;
    long    lastClipTicks;     /* throttle clipboard checks */
    short   initVRefNum;       /* INIT file location */
    long    initDirID;
    Str63   initFileName;
    long    oldFilter;         /* previous jGNEFilter to chain */
    long    lastPollTick;
} Globals;
```

## Boot Sequence

`main()` in `init.c`:

1. `RememberA0` / `SetUpA4` / `DriveRememberA4` — set up A4 for all files
2. `find_reg_base()` — locate extension registers via `SonyVarsPtr`; bail if NULL
3. Shift-key check — skip INIT if Shift held (standard Mac convention)
4. `GetResource('INIT', 128)` → capture file info via `PBGetFCBInfo` → `DetachResource` / `HLock`
5. Allocate `Globals`, `set_globals()` via $020E
6. `kCmdInitIdent` ($0001) — announce API version, INIT version, file location, machine type and system version to host; bail if rejected
7. `InitTrapTables` + install `_HFSDispatch` patch + 22 flat-file trap patches
8. Drain `PollMount` queue → `MountNewDrive` for each pending volume
9. Install jGNEFilter → chain to previous filter

## jGNEFilter

`FilterEntry` in `init.c`, called on every `GetNextEvent`:

1. Save registers, `SetUpA4()`, retrieve `Globals *g`
2. `SyncClipboard(g)` — clipboard sync (internally throttled to every 30 ticks / ~0.5s)
3. `PollMounts(g)` — check for new host-shared volumes, post `diskEvt` if found
4. `PollGuestCmd(g)` — check for host commands: `_Launch` (cmd 1), `ExitToShell` (cmd 2), `ShutDwnPower` (cmd 3)
5. `RestoreA4()`, restore registers, chain to `g->oldFilter`

Mount polling and guest commands run at most once per tick.

## Clipboard Sync

`SyncClipboard` in `clip.c` — bidirectional, per-application:

- **Host → Mac**: Compares the host clipboard sequence number (`ClipSeqNo` $105) against a per-app last-seen value stored on the host via KV ($106/$107). On mismatch, calls `ImportHostToMac` which does `ClipGetLen` ($104) + `ClipImport` ($102) + `ZeroScrap` + `PutScrap`.
- **Mac → Host**: Compares the Mac `ScrapCount` low-memory global against a per-app last-seen value. On mismatch, calls `ExportMacToHost` which does `GetScrap('TEXT')` + `ClipExport` ($101).

Per-app state keying: `appId * 2` for host→mac seq, `appId * 2 + 1` for mac→host scrapCount. This handles MultiFinder's per-partition scrap isolation.

## Trap Patching

Dynamic 68k stubs generated in the system heap by `MakeFlatStub` and `MakeHFSStub`. Each stub saves registers, calls the C dispatch function, and either returns `ioResult` (if handled) or chains to the original trap address (if not ours).

Patched traps:
- `_HFSDispatch` ($A260) — `DispatchHFS`: `GetCatInfo`, `SetCatInfo`, `OpenWD`, `CloseWD`, `GetWDInfo`, `CatMove`, `DirCreate`, `SetVInfo`, `GetFCBInfo`, `GetVolParms`, plus file-ID traps
- 22 flat-file traps via `DispatchFlat`: `_Open`, `_Close`, `_Read`, `_Write`, `_GetVolInfo`, `_Create`, `_Delete`, `_OpenRF`, `_Rename`, `_GetFileInfo`, `_SetFileInfo`, `_UnmountVol`, `_Allocate`, `_GetEOF`, `_SetEOF`, `_FlushVol`, `_GetVol`, `_SetVol`, `_Eject`, `_GetFPos`, `_SetFPos`, `_FlushFile`

Handlers return `kPassThrough` (1) to fall through to ROM for calls not targeting our volume.
